# LAUNCH.md — Ready-to-ship checklist

Fascicle I is feature-complete in the repo. What follows is the list of
**externally-gated** steps that have to happen outside the codebase before
you can flip the switch. Each one is actionable by the maintainer; none
can be automated from inside the repo.

Work through them in order.

---

## 1 · Domain registration

- [ ] Register `praxeos.org` (primary — institutional gravitas fits the
      editorial-brutalist tone).
- [ ] Register `praxeos.xyz` defensively — point it at a 301 redirect
      to `praxeos.org` once the primary is up.
- [ ] Set DNS TTL on both domains to 300s ≥ 24 h before launch so any
      last-minute record changes propagate fast.

Expected cost: ≤ $30/year combined.

---

## 2 · Vercel project + environment variables

- [ ] Create the Vercel project (`william89971/praxeos` as the GitHub
      source).
- [ ] Assign the domains to the project.
- [ ] Set the following env vars in **Production** and **Preview**:

| Key | Value |
|---|---|
| `NEXT_PUBLIC_SITE_URL` | `https://praxeos.org` |
| `NEXT_PUBLIC_MEMPOOL_REST` | `https://mempool.space/api` |
| `NEXT_PUBLIC_MEMPOOL_WS` | `wss://mempool.space/api/v1/ws` |
| `FRED_API_KEY` | **see §4 below** |
| `BLOB_READ_WRITE_TOKEN` | generated by `vercel blob` (for tile storage) |
| `UPSTASH_REDIS_REST_URL` / `_TOKEN` | optional — dev works without |
| `NEXT_PUBLIC_SUPABASE_URL` / `_ANON_KEY` / `SUPABASE_SERVICE_ROLE_KEY` | optional until newsletter ships |

- [ ] `vercel --prod` from a clean branch, or push `main` and let
      GitHub-integrated CI run.
- [ ] Verify the preview URL renders before pointing production DNS.

---

## 3 · Tile pyramid upload to Vercel Blob

The local bake writes to `data/tiles/{z}/{x}/{y}.svg`. For production the
same files need to exist behind a CDN.

- [ ] `npm run bake-tiles` locally to regenerate the pyramid.
- [ ] `npx vercel blob put data/tiles --recursive --access public` (or
      equivalent via the Vercel dashboard / API).
- [ ] Update `src/app/api/tile/[z]/[x]/[y]/route.ts` if needed to 302 to
      the blob URL in production (the dev path can stay reading from
      `data/`).
- [ ] Smoke-test: `curl -I https://praxeos.org/api/tile/0/0/0` returns
      `image/svg+xml` with a long `Cache-Control: immutable`.

---

## 4 · FRED API key (M2 meter)

The Halving Garden's corner widget compares USD M2 expansion to BTC
issuance. Without a key `/api/m2` returns `m2Usd: null` and the widget
renders a greyed-out placeholder.

- [ ] Create an account at <https://fred.stlouisfed.org/docs/api/api_key.html>.
- [ ] Generate a key (free, no card).
- [ ] Set `FRED_API_KEY` in Vercel env vars.
- [ ] Trigger a redeploy; `/api/m2` should now return real numbers.

---

## 5 · Supabase newsletter (optional, can ship later)

The newsletter is behind the `hasSupabase` env-flag. When the envs are
absent the signup form is hidden cleanly. To enable:

- [ ] Create a Supabase project.
- [ ] Apply the migration at `supabase/migrations/20260501000000_newsletter.sql`
      (create the file from the schema in `/docs/ARCHITECTURE.md`).
- [ ] Set `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
      `SUPABASE_SERVICE_ROLE_KEY`.
- [ ] Wire Resend (or similar) for send-confirmation emails if you want
      double-opt-in.

Fascicle I can launch without this.

## 6 · Expert reviewer round

The plan requires a knowledgeable reviewer per module essay before
launch. The emails at `public/press/emails/{ammous,murphy,klein,deist}.md`
are pre-drafted.

- [ ] Send the four emails at least **two weeks** before the public launch.
- [ ] Track replies. Expect at least one correction per essay — bake them
      in.
- [ ] If anyone asks to be credited, add them to `/colophon` under
      *Reviewers*.

Do not launch without at least one expert sign-off on each module.

run them locally as a final check.

- [ ] `npm run test:e2e` — all Playwright specs pass

      `/glossary`. Performance ≥ 0.92, a11y = 1.0, SEO = 1.0.

- [ ] Run `WAVE` against each module. Inspect any contrast warnings.
- [ ] Safari desktop, Safari iOS, Firefox, Chrome Android — each module
      page loads and is interactive.
- [ ] Reader mode in each browser on `/manifesto` and each module essay.
      Must be clean.
- [ ] Print preview of `/manifesto-print` in Chrome: one continuous
      column, no chrome, *Homo agit.* watermark present.

      fall back to its poster.

      focus rings visible.

## 9 · Load test (k6)

Run `k6` against the preview URL before announcing publicly.

```bash
k6 run scripts/k6/tile.js     # /api/tile — 400 req/s for 5m
k6 run scripts/k6/blocks.js   # /api/blocks — 200 req/s for 5m
k6 run scripts/k6/home.js     # / — 200 req/s for 5m
```

(Write the k6 scripts from the plan's §7 if not yet present.) Target:
p95 < 400 ms on the tile route, < 250 ms on home.

---

## 10 · Launch sequencing

When all above are green:

1. Lower DNS TTL to 300 s. Wait 24 h.
2. Schedule `public/press/twitter-thread.md` tweets with 60 s spacing.
3. At 9 AM US-Eastern Tue/Wed/Thu: submit
   `public/press/show-hn.md` to Hacker News.
4. 10 minutes later: post the Twitter thread.
5. 10 AM US-Eastern: send the four pre-drafted emails
   (`public/press/emails/*.md`), using the replies you've already received
   from the reviewer round as social proof.
6. Monitor `/api/m2`, `/api/blocks`, `/api/tile/*` latency for the first
   two hours. Warm the cache for tiles before the HN submission.
7. Do not engage political baiting. Answer technical and economic
   questions.
